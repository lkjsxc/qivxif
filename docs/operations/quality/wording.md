# Wording

## Rules

- Avoid stale product-line labels.
- Avoid numbered generation labels.
- Avoid external palette names in public docs, source, tests, and styles.
- Avoid retired inspiration-project names in public docs, source, tests, and styles.
- Prefer `schema_contract`, `layout_contract`, and `build_contract` for durable formats.
- Remove retired names instead of keeping aliases.
