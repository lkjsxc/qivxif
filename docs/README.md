# qivxif Documentation

`docs/` is the navigation root for qivxif. It defines the product, architecture, repository rules, and acceptance gates.

## Rules

- Docs become true before implementation changes.
- Every directory has one `README.md` table of contents.
- Markdown files stay at 300 lines or fewer.
- Authored source files stay at 200 lines or fewer.
- Retired contracts are deleted, not aliased.
- Durable facts live in one owner doc and are linked from related docs.

## Root Index

- [AGENTS.md](AGENTS.md): docs-specific agent contract.
- [active-work.md](active-work.md): current lane and stop condition.
