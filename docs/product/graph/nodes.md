# Nodes

## Kinds

- text
- markdown
- blog post
- short post
- user profile
- tag
- topic
- tile layout
- pane
- feed
- media
- search view
- graph board
- settings
- system

## Behavior

- Users create and open nodes from the tile shell.
- Node metadata is editable through events.
- Unknown node kinds are rejected until extension docs exist.
- Tombstoned nodes stay inspectable through history.
